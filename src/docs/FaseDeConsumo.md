Fase 3: Arquitectura Frontend (Vue 3) y Experiencia de UsuarioLa capa de presentación es el punto crítico para la adopción del sistema. El código proporcionado estructura una base sólida con Vue 3 (Composition API) y Tailwind CSS. Sin embargo, para escalar este prototipo a un producto de grado de producción que emule la fluidez de plataformas comerciales modernas, es imperativo modularizar la interfaz y centralizar el manejo de datos.A continuación, la orquestación arquitectónica para el consumo de la API y la reactividad del frontend.1. Gestión de Estado Global (Pinia)El ecosistema actual muestra la inyección de useAuthStore. Para la gestión de la cartelera, se requiere implementar un useEventStore.El enfoque tradicional: Cada componente (el calendario lateral, el carrusel de destacados y el feed de próximos eventos) realiza peticiones HTTP independientes al backend (FastAPI) al montarse en el DOM. Esto duplica el consumo de red, satura el servidor y genera desincronización visual.La solución arquitectónica: Centralizar la petición en Pinia. El store invoca GET /api/v1/events una sola vez y almacena el arreglo de datos en memoria.Valor para el negocio: A partir del estado global, se derivan propiedades computadas (computed). El carrusel consume featuredEvents (filtrado por el mayor score) y el widget lateral consume calendarEvents (mapeado por fecha). Si un usuario marca un evento como favorito, el store muta el estado localmente, reflejando el cambio al instante en todas las vistas sin necesidad de recargar la página, logrando una experiencia de fricción cero.2. Modularidad y Control de Acceso en Interfaz (RBAC)El archivo monolítico proporcionado debe segmentarse en componentes atómicos para facilitar el mantenimiento y la inyección de la lógica transaccional.Componente <EventModal> (El Centro de Operaciones): Se activa al hacer clic en un <EventCard>. Este modal no solo muestra los detalles extendidos, sino que actúa como el panel de administración descentralizado.Lógica Comparativa de Seguridad: En lugar de ocultar visualmente botones con CSS (una práctica insegura), el botón de "Editar/Eliminar" se renderiza en el DOM de forma condicional (v-if="authStore.user.id === event.owner_id").Resultado Operativo: El usuario interactúa únicamente con la información que le compete, mitigando errores operativos y simplificando la interfaz visual.3. Flujo de Publicación y Optimización de ImágenesEl botón "Proponer Evento" detonará un formulario modal. El manejo de la imagen representativa del evento es el mayor cuello de botella en aplicaciones web.Método tradicional: Convertir la imagen cargada por el usuario a una cadena Base64 e inyectarla en el payload JSON. Esto multiplica el peso de la petición y bloquea el hilo principal del navegador.Solución eficiente: El componente de formulario captura el archivo File nativo. Mediante la API FormData, se envía directamente al servidor (o se utiliza una URL pre-firmada para subirla directo al proveedor de almacenamiento en la nube). FastAPI procesa y devuelve la URL estática.Valor para el negocio: El renderizado de la imagen en la tarjeta del evento (<img :src="event.image_url">) se delega a una Red de Entrega de Contenidos (CDN). Esto acelera los tiempos de carga del frontend, factor crítico para la visualización fluida tanto en dispositivos móviles como en los tótems interactivos proyectados.4. Despliegue en Producción y CI/CD (Vercel)Para llevar esta Single Page Application (SPA) en Vue 3 a producción, el entorno debe configurarse para maximizar la disponibilidad y el Posicionamiento en Buscadores (SEO).Infraestructura: El despliegue de la interfaz estática compilada (npm run build) se orquestará sobre la infraestructura perimetral (Edge Network) de plataformas como Vercel.Configuración de Enrutamiento: Dado que Vue Router maneja la navegación en el cliente, es obligatorio configurar reglas de reescritura en el servidor de producción (ej. vercel.json con "rewrites": [{"source": "/(.*)", "destination": "/index.html"}]). Esto previene errores HTTP 404 cuando los usuarios acceden directamente a la URL de un evento específico o recargan la página.Integración Continua: La conexión del repositorio Git con la plataforma de alojamiento habilitará despliegues automáticos (CI/CD). Cada cambio fusionado en la rama principal construirá y distribuirá la nueva versión de la interfaz en minutos, asegurando iteraciones ágiles y controladas.
